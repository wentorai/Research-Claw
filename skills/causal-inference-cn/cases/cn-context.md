# 中国情境通用注意事项

本节列出在中国数据上做因果识别时反复出现的几类陷阱与回应套路。所有方法的执行都应过一遍此清单。

## 1. 中国数据可得性（主要数据库特点）

中国微观经验研究的数据来源相对集中，每个数据库各有侧重与陷阱：

- **CSMAR（国泰安）**：A 股上市公司财务、治理、市场数据；行业代码常用证监会 2012/2017 双版本，长期面板需手工对齐编码。变量命名同一字段在不同表中可能不一致（如 "TotalAssets" vs "TA"），合并前先 codebook 比对。
- **Wind / 同花顺**：公司公告、研报、宏观指标更新最快，是事件研究的首选。但部分历史数据有静默修订，导出时务必带"数据快照日"。
- **CFPS（中国家庭追踪调查）**：北大开放微观追踪数据库，2010 始隔年，社会学/劳动经济学常用；样本权重设计要严格使用 `svyset`（Stata）或 `survey`（R）。
- **CHFS（中国家庭金融调查）**：西财开放，2011 始，金融行为重点。访问需申请，但数据字典齐全。
- **CMDS（中国流动人口动态监测）/ CHARLS（健康与养老追踪）**：劳动 / 健康经济学补充。
- **统计年鉴 / EPS / 中经网**：地级市、县级面板首选；县级数据中"市辖区合计"与单独"区"易重复。
- **海关 / 税务 / 工商**：受限，多以购买或合作形式获取。

关键陷阱：变量定义不连贯（"营业收入" vs "主营业务收入"在 2007 新会计准则前后口径不同）；样本选择性偏（CSMAR 仅含上市，结论不能外推到非上市企业）。

## 2. 行政区划层级匹配陷阱

中国行政区划（省/市/县/区）在研究期内频繁变动：

- **2010-2024 年间**约 100 余次县改区、地级市撤设、县级市改名、街道办合并。直接用区划码合并面板会出现"消失的县"和"重复的区"。
- **国家统计局区划码**（GB/T 2260）每年发布新版，年内调整不进当年版本。建议固定在某一年（如 2020 年版）做跨年合并。
- **市辖区合计 vs 各区**：地级市数据中常出现"XX 市市辖区"作为汇总行，与下属各区重复，求和会双计。
- **"地区"概念混淆**：CSMAR / Wind 中"注册地"与"办公地"可能不同（如阿里注册杭州，办公地分散）；个人数据则有户籍地 vs 居住地。
- **解决方案**：维护一张随时间变动的"区划映射表"（reference table），每次合并显式 `merge` 检查 `_merge` 取值；建议使用国务院发改委发布的"行政区划变更专题"作权威源。

## 3. 国家政策外生性的常见质疑与回应

中国研究最常被审稿人质疑的是"政策不外生"——政策对象本身就是因为某些原因（经济基础好/差）才被选中：

- **试点政策的选择性**：碳交易、自贸区、智慧城市试点等首批名单往往不是随机；批方有意挑能成功的城市做。
- **回应套路**：
  1. **制度背景叙述**：详细引用政策文件，说明遴选标准是否依赖待研究 outcome 的事前值；如果依赖，转向 IV 或 SC。
  2. **PSM-DID**：先按试点选择标准做 PSM，再 DID。是中国会计/管理/区域经济论文的标配。
  3. **平行趋势 + 安慰剂时间**：把假处理时点提前 2-3 年，看是否有"提前效应"；若有，政策可能内生。
  4. **遗漏变量稳健性**：Oster (2019) δ 检验，量化未观测混淆的容忍度。
  5. **改用 SC**：单一处理单位时，SC 对外生性要求更弱（仅需"未受冲击"控制池）。
- **不要写**："政策由中央决定，因此外生"——这一论点不被认真对待。

## 4. 中国学者常用 STATA 包列表

这些包构成中国经济学、金融学、会计学论文的"事实标准"工具链：

- **`reghdfe`** — 高维固定效应（行业 × 年份 × 省份）必装；速度比 `xtreg` 高一个数量级
- **`ivreghdfe`** — `reghdfe` + 工具变量
- **`winsor2`** — 缩尾处理（默认 1% / 99%），上市公司数据几乎所有连续变量都需要
- **`asdoc`** — 一键导出回归表为 Word/Excel，国内学位论文标配
- **`estout` / `esttab`** — 论文级 LaTeX 表格，国际期刊投稿首选
- **`xtset`** + `xtreg` / `xtologit` — 面板基本操作
- **`psmatch2` / `pstest`** — PSM 与平衡检验
- **`csdid` / `did_imputation` / `eventdd`** — 现代 DID
- **`rdrobust` / `rddensity`** — 现代 RDD
- **`synth` / `synth_runner` / `sdid`** — 合成控制
- **`bootstrap` / `boottest`** — 小样本 / 少 cluster bootstrap
- **`oster`** — Oster (2019) 遗漏变量敏感性
- **`coefplot`** — 系数图（事件研究、异质性）

工作流惯例：缩尾 → 主回归 (reghdfe) → 异质性 (split sample / 交互项) → 稳健性 (PSM-DID, IV, SC) → 机制检验 (中介效应或调节效应)。

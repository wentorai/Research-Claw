# 政策评估典型套路（含假想案例）

本节给出"中国政策评估"的标准 SOP，并用两个**完全假想**的案例演示如何把通用因果识别方法应用到具体政策。不引用任何真实未发表研究。

## SOP（5 步）

1. **政策刻画**：政策文本、时点、覆盖范围、强度（连续 vs 二元）、是否分批推开。
2. **识别策略选择**：
   - 多单位、多时点、面板可得 → **DID** (优先 csdid)
   - 单一处理单位 + 长 pre 期 → **SC**
   - 阈值规则（人口/规模/分数） → **RDD**
   - 政策外生性可疑 + 找到合适工具 → **IV**
   - 横截面观测 + 强观测协变量 → **PSM/DR**（仅作辅助）
3. **数据构造**：处理变量定义清楚（强度型 vs 二元）、时间窗口对齐、样本筛选规则透明。
4. **主回归 + 平衡 + 稳健性 + 机制**：每一步都对照 `cases/robustness-checklist.md`。
5. **正面回应外生性质疑**：见 `cases/cn-context.md` § 3。

## 假想案例 A：某省"绿色金融激励政策"对企业 R&D 投入影响

> 完全虚构，仅用于演示方法套用。

**设定**：假设 2018 年某省（设为 X 省）在全省范围内推行"绿色金融激励政策"——对节能减排达标企业贷款利率下调 50 BP。我们想评估：政策是否提高目标企业 R&D 投入？

**识别策略**：单一处理单位（X 省）→ **SC** 为主。

**数据**：CSMAR 上市公司 2010–2022 年面板，因变量 lnR&D，控制变量含 lnAsset, leverage, ROA, age；按省份汇总到 31 省 × 13 年面板。

**步骤**：

1. **基础 SCM**：以 X 省 lnR&D 为目标，其余 30 省为控制池；预测变量包括 GDP 增速、第二产业占比、专利申请数；time.optimize.ssr = 2010-2017。
2. **Pre-RMSPE 检查**：若 < 0.05，拟合 OK；> 0.10 转 Augmented SCM (`augsynth`).
3. **In-space placebo**：把每个非 X 省作假处理，得到 30 个 placebo gap 路径；真实 X 省 gap 应在分布尾部。
4. **稳健性**：
   - Leave-one-out（剔除权重最大的某省，看主路径是否变）
   - 改预测变量集（添加/移除人均 GDP）
   - Synthetic DID（`synthdid`）作为补充
5. **结果叙述**：报告处理后年份 ATT 路径 + 总累积效应；明确 LATE 解读（"对 X 省 + 受激励企业"，不外推到全国）。

**回应"内生性"**：制度背景上，X 省入选并非随机——叙述时需说明是中央指令而非省内主动申请；若仍可疑，**改用 IV**：以"是否被中央点名"为 Z，"省级出台细则"为 D。

## 假想案例 B：某市"营业额规上企业"数据上报阈值的"规模操纵"效应

> 完全虚构，仅用于演示方法套用。

**设定**：假设某市规定营业收入 ≥ 2000 万元的企业必须按月上报全口径财务数据（"规上企业"），低于此阈值仅需季报。我们想评估：阈值是否导致企业在 2000 万附近"凑数"以越过阈值（操纵效应）？以及越过阈值是否对企业实际经营有影响？

**识别策略**：阈值规则 → **RDD**。但因变量"是否操纵"本身需先做 **McCrary 密度检验**判断阈值附近是否有 bunching。

**数据**：工商年度报表 2015–2020，N ≈ 50 万家企业。Running variable X = 营业收入 - 2000 万。

**步骤**：

1. **第一图**：`rddensity` 检验 X 在 0 附近密度是否连续。**预期看到右侧密度跳跃**（操纵证据）；若如此，**RDD 在因变量"实际经营"上不再有效**（个体在精确操纵 X）。
2. **若密度连续（无操纵）**：才能继续做 sharp RDD：把"上报频率 = 月报"作为处理 D，因变量"研发投入"或"税务合规度"作为 Y，做 `rdrobust`.
3. **若密度不连续**：转 **bunching estimator** 估计操纵规模（Saez 2010, Chetty et al. 2011），不再继续做 RDD。
4. **协变量平衡**：在阈值附近行业、年龄、所有制结构应连续。
5. **多带宽稳健**：MSE-optimal h、0.5h、2h。
6. **解读限制**：即使 RDD 有效，识别的也是"刚跨阈值企业"的局部效应，不能外推到大型规上企业。

**回应"阈值多重处理"**：阈值同时触发**多个**政策（统计上报、税收审计、社保稽核）→ τ 是这些政策的**捆绑效应**，不是"上报频率"的单独因果效应。论文标题与抽象描述都需明确这一点。

## 通用警示

- **处理强度**：连续型政策强度（如"补贴强度"）比二元型更易识别，但需要可信的强度测度数据。
- **空间溢出**：政策受益方与对照方地理临近时，对照单位被"挤入"或"挤出"——SUTVA 失效。常用做法：剔除处理单位的相邻县/市作对照，或显式建模溢出效应。
- **政策叠加**：研究期同时存在其他政策时（如双碳叠加供给侧改革），需要在控制变量纳入或在样本上避开。
- **审稿人最爱问**：（1）你的对照组为什么真的可比？（2）有没有可能是别的同时变化的因素？（3）样本到 2020 年你怎么处理疫情？

# Prompt: 创新点撰写

## 用途
帮助用户撰写各类项目的"创新点"段落。**创新点是评分权重最高的指标之一**，但也是最容易写得平庸的部分。

## 项目体例映射

| 项目 | Section 名 | 篇幅指引 |
|------|-----------|---------|
| 国家社科基金 | "本课题的特色与创新之处" | ~500-1500 字 |
| NSFC（互补 Skill） | "本项目的特色与创新之处" | ~1 页 |
| NIH R01 | Innovation（独立 subsection） | ~1 page |
| NSF | 散布在 Project Description 内 + 体现在 Intellectual Merit | ~0.5-1 page |
| ERC | Extended Synopsis "ground-breaking nature" 段 + B2 中段 | 0.5-1 page synopsis 内 |
| MSCA PF | Excellence section 的 "originality and innovative aspects" | ~0.5 page |

## 创新点的三种类型（需明确归类）

强建议在创新点段落开始就**明确归类**到下列至少一种：

1. **理论 / 概念创新**（conceptual / theoretical innovation）
   - 提出新假设、新框架、新概念
   - 重新定义已有概念
   - 跨学科理论整合
2. **方法 / 技术创新**（methodological / technical innovation）
   - 新研究方法 / 实验技术
   - 新工具 / 新算法
   - 新数据源（自建调查 / 行政数据 / 新仪器）
3. **应用 / 转化创新**（applied / translational innovation）
   - 基础到临床 / 政策的桥梁
   - 跨场景 / 跨人群 / 跨地域的迁移
   - 工程 / 产品化的可行路径

**反例**：只笼统写"本研究具有理论与应用双重创新" — 必须具体到哪种 + 怎样新。

## 输入要求

询问用户：
1. 目标项目类型
2. **声明的创新点**（用户初步想法）
3. **与最相关的 3-5 篇前人工作的对比**（specific comparator）
4. **是否有 preliminary data 支撑创新可行**

## 输出模板

### 国家社科 / NSFC 体例

```
本课题的特色与创新之处主要体现在以下三个方面：

1. **理论创新**：[具体到提出 / 修正 / 整合何种理论]
   - 与 [代表性文献 / 学派] 的对话点
   - 我们的不同主张

2. **方法创新**：[具体到使用何种新方法 / 数据源]
   - 数据 / 方法的特殊性（如自建 panel、行政数据、新计量识别策略）

3. **观点 / 视角创新**：[具体到从何种新视角切入]
   - 与既有视角的差异
   - 预期得出的新论断
```

### NIH Innovation 体例

```
INNOVATION

This proposal is innovative in three respects:

1. **Conceptual innovation**: We challenge the prevailing assumption that 
   [既有共识]. Our central hypothesis posits [新观点], which has not been 
   tested in the context of [scope].

2. **Technical innovation**: We will leverage [新技术 / 新模型], which 
   provides [advantage] over existing [comparator] approaches. Our 
   preliminary data (Fig. X) demonstrates feasibility.

3. **Translational innovation**: By bridging [基础发现] and [临床问题], 
   our work has the potential to inform [具体临床路径].
```

### NSF / ERC 体例（强调 ground-breaking）

```
[ERC] The ground-breaking nature of this project lies in:

(i) [核心新假设 / 概念]
    Current research assumes [现状]; we propose [新假设], leveraging recent 
    advances in [enabling technology / theory].

(ii) [方法学突破]
     We will develop [novel methodology] that overcomes [既有方法 limitation].

(iii) [跨学科融合]
      By integrating [学科 A] and [学科 B], we open a new line of inquiry 
      previously inaccessible.

The high-risk-high-gain element is that [具体风险]; if successful, the 
project will [transformative outcome].
```

## 创新点的常见低分写法

| 低分写法 | 为何低分 |
|---------|---------|
| "本研究综合运用 A、B、C 多种方法" | 综合 = 工作量大，但不是创新 |
| "首次系统研究 XX" | 不论证为何之前没人做（因为没价值？）就喊"首次" |
| "国内空白" | 国内空白 ≠ 学术创新 |
| "本研究将填补 XX 领域研究空白" | 太空泛，没说明 gap 是什么 |
| "与 [作者] 不同，我们 ..." 但没说明哪里不同 | 缺 specific comparator |
| 把"丰富了 XX 理论"作为创新点 | "丰富" 不是论断 |

## 创新点的高分写法（共通要素）

1. **可对比**：每个创新点必须有 specific comparator（哪些工作 / 哪些假设）
2. **可证伪**：创新点能转化为可检验的假设
3. **可承担**：申请人 / 团队 / 设备能落地（与可行性自洽）
4. **可量化**：尽量有量化指标（如 effect size、accuracy improvement、coverage extension）

## 工作流

1. 询问用户目标项目 + 输入要求
2. 检查每个 claimed innovation 是否有 specific comparator；缺则反追问
3. 按目标体例输出 1 个版本
4. 提醒用户用"工作基础 / preliminary data"反向印证：如果创新点说"提出 X 框架"，preliminary 是否已经有 X 的雏形？
5. 把"创新点"与"研究内容 / 研究方案"做一致性检查 — 避免创新点空喊但方案没承接

## 自检清单

每个声明的创新点逐条问：

- [ ] 是哪种类型的创新（理论 / 方法 / 应用）？
- [ ] 与谁对比？specific 引用了吗？
- [ ] 是否可证伪？
- [ ] 是否在研究方案中有相应 sub-aim 实现？
- [ ] PI 团队 / 工作基础是否撑得起？

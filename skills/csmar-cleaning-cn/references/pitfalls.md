# 国内学者高频陷阱

实证经管研究中**最常被审稿人挑出**或**最常自己踩坑**的样本处理问题。每条都包含：症状 → 诊断 → 修复。

---

## 陷阱 1：金融业剔除不彻底

**症状**：杠杆率 (Lev) 的均值异常高（>1）、研究内生性的论文系数被极端值绑架。

**诊断**：`tab IndCd` 看是否仍有 `J*` 开头的观测。

**常见原因**：
- 用 CSRC 2001 编码剔除（剔了 `I`），但数据里有部分 CSRC 2012 编码 (`J`) 没剔
- 用了 `IndCd == "J"` 做精确匹配（漏掉中类 `J66`、`J67` 等）
- 未剔除"类金融"业务（信托、租赁、私募、互联网金融等）

**修复**：
```stata
drop if substr(IndCd, 1, 1) == "I" | substr(IndCd, 1, 1) == "J"
```

---

## 陷阱 2：把 `Stkcd` 当作不变的公司 ID

**症状**：面板中某家公司 ROA、营业收入等指标在某一年突然跳变 10 倍以上；CEO 任期的累积量异常高。

**诊断**：`bys Stkcd: gen pct_change = (revenue - revenue[_n-1]) / revenue[_n-1]` 后查看 `pct_change > 5` 的样本。

**根因**：借壳上市，`Stkcd` 不变但公司主体已变（详见 `id-tracking.md` 情形 3）。

**修复**：
- 引入 CSMAR 重大资产重组数据库标记借壳事件
- 将借壳前后视为两家公司，或剔除借壳过渡年

---

## 陷阱 3：缩尾时机错误

**症状**：稳健性检验（剔除小样本子集后重新回归）系数符号变化巨大，无法解释。

**根因**：在子样本里**重新缩尾**，导致主回归与稳健性回归的极值定义不同。

**修复**：
- 全样本一次性缩尾，**子样本回归不再二次缩尾**
- 主回归与所有稳健性回归共享同一份缩尾后的样本

---

## 陷阱 4：忽视年报披露时滞导致 look-ahead bias

**症状**：用 t 期财报预测 t 期收益的回归 R² 异常高（如 > 0.5），但 t+1 收益预测就崩溃。

**根因**：t 期年报在 t+1 年 4 月才披露，但回归用的是 t 期 1–4 月的收益（彼时年报尚未公布）。这是**未来信息泄漏**。

**修复**：
```stata
* 收益从 t+1 年 5 月开始用 t 期年报
gen yr = year(Trddt)
gen mn = month(Trddt)
gen merge_year = cond(mn <= 4, yr - 2, yr - 1)
* 保险做法：用披露日 DeclareDate
```

详见 `time-alignment.md` 关键认知 3。

---

## 陷阱 5：面板不平衡当作平衡处理

**症状**：`xtreg, fe` 报告的"组间差异"系数无法重现。

**根因**：A 股面板**天然不平衡**（IPO 时点不同、退市、暂停上市）。如果按平衡面板的逻辑构造（如用 `unbalanced=False`），会损失大量观测。

**修复**：
- 接受不平衡面板（`xtreg` 默认支持）
- 报告 `xtdescribe` 输出的"observations / firm" 分布
- 不要为了平衡而强行剔除"观测年数 < N"的公司——这是样本选择偏差

---

## 陷阱 6：把"剔 ST"用简称匹配

**症状**：剔除后还有 ST 公司的观测；或正常公司被误剔。

**根因**：用 `strpos(Stknme, "ST") > 0` 匹配会：
- 漏掉早期还没标记为 ST 但 `Trdsta` 已是 ST 状态的样本
- 误剔含 "ST" 字符串的非 ST 公司

**修复**：
```stata
* 优先用 Trdsta 字段（数值码精确）
drop if Trdsta != 1
* 简称匹配作为备选
```

---

## 陷阱 7：混合使用合并报表与母公司报表

**症状**：合并财务面板时观测数翻倍。

**根因**：CSMAR 财报表的 `Typrep` 字段：
- `A` = 合并报表 (Consolidated)
- `B` = 母公司报表 (Parent only)

如果不筛选 `Typrep`，每个 Stkcd-Accper 会有两条记录。

**修复**：
```stata
keep if Typrep == "A"  // 一律用合并报表（公司金融研究主流）
```

---

## 陷阱 8：研发支出 NaN 默认填 0

**症状**：研发投入 (R&D) 与公司绩效的回归显示研发越高绩效越差，与文献相反。

**根因**：早期年份（2007 年前）许多公司不披露 R&D，CSMAR 字段为 NaN。如果直接 `df.fillna(0)`，会把"不披露"误判为"R&D = 0"，使得真实 R&D 高的公司反而看起来与"假 0"公司接近。

**修复**：
- **不要**对 R&D 缺失值填 0
- 限定样本期：从 R&D 强制披露年份（约 2018 年后）开始
- 或仅保留 R&D 非空的子样本，并明示"自我选择"问题

---

## 陷阱 9：行业固定效应粒度过细

**症状**：FE 吸收掉关键变异，主自变量不显著。

**根因**：用了申万三级行业 (60+ 个) + 年（20 年）= 1200+ 个 FE，对小样本（< 30000 观测）会让每个 cell 只有 20 来个观测，吸收过量变异。

**修复**：
- 默认用 CSRC 中类（约 80 个）
- `IndCd#year` 仅在样本量 > 50000 时才用
- 报告不同 FE 粒度下的稳健性

---

## 陷阱 10：把"暂停上市"等同于"退市"

**症状**：研究公司风险预测时，把暂停上市归入"退市"事件，导致预测准确率虚高。

**根因**：暂停上市后可恢复（`Trdsta` 中暂停码 ≠ 退市码）。

**修复**：
- 用 `Delisting_Date` 字段精确判断退市
- `Trdsta` 中的暂停状态视为"高风险但仍存活"

---

## 陷阱 11：误用市值做规模代理

**症状**：Size = ln(Market Cap) 的回归系数不稳定。

**根因**：A 股市值受流通股 / 总股本差异影响巨大（特别是 2007 年股权分置改革前后）。**市值代理稳健性低**。

**修复**：
- **优先用 ln(总资产) 作 Size**（财务指标稳健）
- 市值仅作稳健性
- 注意 CSMAR 字段：`Dsmvosd`=流通市值，`Dsmvtll`=总市值

---

## 陷阱 12：ROA 分子分母期间不匹配

**症状**：ROA 分布异常宽，部分公司 ROA > 1 或 < -1。

**根因**：
- ROA = 净利润 / 总资产
- 净利润是**当期累计值**，总资产是**期末时点值**
- 标准做法：分母用 `(期初 + 期末) / 2` 平均资产

**修复**：
```python
# 期初资产 = 上一期年报的资产总计
df = df.sort_values(["Stkcd", "year"])
df["assets_begin"] = df.groupby("Stkcd")["A001000000"].shift(1)
df["assets_avg"] = (df["A001000000"] + df["assets_begin"]) / 2
df["ROA"] = df["B002000000"] / df["assets_avg"]
```

> 部分论文偷懒用期末资产，可接受但应在脚注说明。

---

## 陷阱 13：聚类标准误粒度不当

**症状**：t 值高得离谱（10+），不可信。

**根因**：
- 没有聚类（`vce(robust)` 而非 `vce(cluster Stkcd)`）
- 只在公司层聚类没在年份聚类（残差有时间相关）

**修复**：
- 默认 **公司层聚类**：`vce(cluster Stkcd)`
- 时间维度长（>10 年）+ 截面相关明显（如政策冲击）：**双向聚类** `vce(cluster Stkcd year)`（用 `reghdfe`）

---

## 陷阱 14：稳健性检验做"反向"操作

**症状**：稳健性"剔除头部公司"反而强化主回归结论，被审稿人质疑挑选样本。

**修复**：稳健性应**朝两个方向**报告（剔除头部 + 仅保留头部），并解释结果差异。

## Quick checklist（投稿前自检）

- [ ] 金融业剔除（CSRC 2001 `I` + 2012 `J`）
- [ ] ST/*ST/PT 剔除（用 `Trdsta` 而非简称）
- [ ] IPO 当年剔除
- [ ] B 股剔除（首位 `2`/`9`）
- [ ] 关键变量缺失值删除（不填 0、不插补）
- [ ] 缩尾在最终样本上一次性做（1%/99%）
- [ ] 合并报表（`Typrep == "A"`）
- [ ] 财报披露时滞已处理
- [ ] 公司层聚类标准误
- [ ] 行业 + 年双向 FE（`reghdfe`）
- [ ] xtdescribe 输出贴附录或脚注

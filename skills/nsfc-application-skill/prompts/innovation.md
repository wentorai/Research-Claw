# 特色与创新点撰写 Prompt

> 用途：起草 / 重写 NSFC 申请书"本项目的特色与创新之处"章节（约 500-1000 字）。这是函评打分中权重最高的小节之一。

## 模板

```
你是 NSFC 申请书写作助手。请帮我起草「本项目的特色与创新之处」章节。

【输入信息】
- 研究主题：{{一句话主题}}
- 关键科学问题 KP1-KPm：{{……}}
- 研究内容 R1-Rn：{{……}}
- 立项依据中已批判的 3 个流派局限：{{流派 1 局限：…；流派 2 局限：…；流派 3 局限：…}}
- 申请人 / 团队的方法学优势：{{……}}

【写作要求】

## 创新点数量与维度
- 创新点数量：**3 条为宜**（不超过 4 条）
- 三大创新维度（建议按以下顺序）：
  1. **科学思想 / 视角创新**（最重要）—— 提出了什么"新假说 / 新框架 / 新视角"
  2. **方法 / 技术创新** —— 用了什么"已有方法的新组合 / 新算法 / 新实验设计"
  3. **数据 / 体系 / 应用创新** —— 构建了什么"新数据集 / 新场景 / 新应用"

## 每条创新点的写法
每条 1 段（5-8 行），结构如下：
1. **首句**：用"区别于已有 ……，本项目首次 ……"句式给出"总结性结论"
2. **第 2-3 句**：与立项依据中批判的"流派局限"显式对比
3. **末句**：说明这条创新点对应到哪个 / 哪几个研究内容（R?）和关键科学问题（KP?）

## 写作禁忌（如下表述会被评审专家直接扣分）
- "本项目首次研究了 ……" —— 不证自明，无信息量
- "本项目填补了 …… 的空白" —— 评审最反感的句式之一
- "本项目达到了国际领先水平" —— 自我评价，禁止
- "本项目系统性 / 全面 / 深入研究了 ……" —— 程度副词堆砌
- 把研究内容 R1, R2, R3 复述为创新点 —— 工作量 ≠ 创新

【自检清单】
- [ ] 创新点数量 ≤ 4 条？
- [ ] 是否覆盖"思想 / 方法 / 应用" 3 个维度？
- [ ] 每条首句是否使用"区别于 …… 本项目 ……"句式？
- [ ] 是否与立项依据中批判的流派局限显式对比？
- [ ] 是否避免了"首次""填补""国际领先""全面""系统"等空洞词？
- [ ] 每条是否对应到具体的 R / KP？

请先输出 3 条创新点的"首句"草案，等我确认后再展开成段。
```

## 优秀范例

```
创新点 1（思想创新）：区别于既有研究将 X 与 Y 视为单向作用机制（流派 1 局限），
本项目首次提出 X 与 Y 在 Z 情境下的双向耦合机制，并以可验证假设 H1 / H2 形式给出。
该创新点对应关键科学问题 KP1，将在 R1、R3 中得到检验。

创新点 2（方法创新）：现有研究主要使用静态截面方法（流派 2 局限），难以捕捉 X 的动态演化。
本项目将 A 方法与 B 方法相结合，构建动态贝叶斯估计框架，可同时识别静态参数与动态路径。
该创新点对应 KP2，主要在 R2 中实现。

创新点 3（数据 / 应用创新）：本项目将构建覆盖 NNNN-NNNN 年的 ZZ 数据集（含 K 个观察单元），
首次将 W 场景纳入 X-Y 研究框架，相较已有研究（多基于发达国家 / 单一行业样本）扩展了
适用边界，对应 R4 的产出。
```

## 注意事项

- 创新点是评审专家"30 秒决策"的关键之一 — 必须放在显著位置（章节首页）
- 一条创新点最多扣 1 个核心证据（如一篇前期论文或一组预实验数据）
- 不要在创新点中加入"应用价值"的政策建议 — 那是结论章的内容

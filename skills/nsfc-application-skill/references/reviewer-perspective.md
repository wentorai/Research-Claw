# NSFC 评审专家视角：函评 + 会评（深度版）

NSFC 实行 **"函评 + 会评"双轮制度**。理解评审视角是申请书写作的第一性原则——写作目标不是"打动自己"，而是"通过 5 位小同行函评 + 学科组会评的双重过滤"。

> 本文件基于 NSFC 公开发布的《项目评审办法》《评审专家行为规范》、各学部公开发布的"年度评审工作综述"（《中国科学基金》期刊）以及多位获资助专家在公开讲座 / 公开博客中的分享整合而成。具体打分规则、上会比例每年微调，**最终请以当年公开文件为准**。

---

## 一、函评（通讯评议）

### 1.1 评审主体

- 由 NSFC 科学处从专家库中按 **学科代码 + 关键词 + 研究兴趣** 匹配派送
- **每份申请书派送 5 位** 小同行（地区项目 / 青年项目至少 3 位）
- 评审为匿名、网络化操作（在 NSFC ISIS 系统内完成）
- 评审专家信息全程对申请人保密；申请人和评审人之间存在"回避关系"由系统自动屏蔽

### 1.2 评审打分（公开制度，2 维 × 5 项）

NSFC 函评打分实际上是 **"综合评价 (4 档)" + "资助意见 (3 档)" 双维结构**——很多写作指南只讲单维度，是不准确的。

#### 维度 A：综合评价（A / B / C / D 4 档）

衡量"申请书的整体学术质量"，专家从以下 5 个分项给出综合判断：

| 分项 | 含义（评审专家心中的具体话语） |
| --- | --- |
| **1. 科学价值 / 学术意义** | "这个问题本身值不值得做？解决了能推动学科 / 满足国家需求吗？" |
| **2. 创新性** | "和已有工作相比，新意在哪？是真新还是伪新？" |
| **3. 研究方案的可行性与合理性** | "技术路线、方法、数据合不合理？预实验有没有？写得空不空？" |
| **4. 申请人 / 研究团队基础** | "这个人 / 这个团队能不能做成？代表作是不是和方向连续？" |
| **5. 预算合理性** | "钱够不够、多没多、用得对不对？" |

- A = 优 ≈ 强烈推荐
- B = 良 ≈ 推荐资助
- C = 中 ≈ 可不资助 / 中性偏负
- D = 差 ≈ 明确反对

#### 维度 B：资助意见（A / B / C 3 档）

专家在综合评价之外，**单独给出资助决策**：

- A = 优先资助
- B = 可资助
- C = 不予资助

#### 总分计算（来自公开博客整合，非官方公布的精确公式，仅作经验参考）

经验性总分 = 综合评分 + 资助建议

- 综合评分 = (A×4 + B×3 + C×2 + D×1) / 评议份数  → 满分 4
- 资助建议 = (A×4 + B×2) / 评议份数  → 满分 4

经验阈值：

- **总分 > 5（约 4 个 A/B 以上）→ 一般可上会**
- **总分 < 4 → 基本退稿**
- 4–5 之间 → 学部根据资助率"在线划线"

> 注：此公式为坊间整理的"等效计算"，不同学部、不同年份的具体加权细节有调整。**重要的不是记住公式，而是记住"4 个 A 是上会的安全线"**。

### 1.3 评审等级（坊间常用 5 档表述）

部分专家会按以下 5 档作综合标记：

- **A. 优先资助**（综合评价 A + 资助 A）
- **B. 可资助**（综合 B + 资助 B）
- **C. 可不资助**（综合 B/C + 资助 B/C）
- **D. 不予资助**（综合 C + 资助 C）
- **E. 未达基本要求 / 撤回**（学风 / 形式审查问题）

**5 份评审中，通常需要 3-4 份 A/B 才能进入会评。1 份 D 几乎"一票否决"——会评学科组会重点讨论"为什么有 D"。**

## 二、函评专家的真实阅读模式（"60 秒决定生死"内幕）

### 2.1 平均阅读时间

- 每位评审专家会**集中拿到约 15-25 份**申请书，要求约 **2 周内完成**
- 据基金委公开渠道与多位专家访谈整合：**约一半的专家在 2 小时内完成所有标书的"快速分类"**，再花 1-3 天对"待定"的标书做深读
- **真正的精读时间：每份 30-90 分钟**；但**第一遍快速判断只用 3-10 分钟**

### 2.2 "60 秒决定生死"——专家的 4 类文件夹分类

来自坊间总结，函评专家拿到一份申请书后，往往按以下 4 类心理"文件夹"快速归类：

| 文件夹 | 触发条件 | 心理结果 |
| --- | --- | --- |
| **#1（推荐）** | 30 分钟内能读懂 + 打分点清晰（科学问题、创新点、方案 3 处都"一击命中"） | A / B（候选优先） |
| **#2（待定）** | 不能完全读懂，但打分点清晰；或者读懂了但有疑虑 | B / C（看其他几人意见） |
| **#3（边缘）** | 能读懂但打分点模糊（不知道核心创新是什么） | C |
| **#4（毙）** | 既读不懂、打分点也不清晰 | D |

**关键启示**：评审专家不是"读完整本书再打分"，而是"先打分、再读关键支撑"。**前 60 秒的命中决定整个申请书命运**。

### 2.3 评审专家眼睛会本能扫的 6 个位置（按顺序）

1. **题目** —— 是否点题、是否避免空泛
2. **摘要前 2 句** —— 答出"做什么 + 为什么有意义"了吗？
3. **立项依据首页** —— 国内外现状是不是"批判性"组织、研究问题有没有"自然涌现"
4. **创新点章节** —— 是不是 ≤ 3 条、有没有用"区别于已有……本项目……"句式
5. **技术路线图** —— 有没有图、节点是否对应研究内容、有没有信息量
6. **申请人代表作前 3 篇** —— 是不是和课题方向连续

如果这 6 处中任何一处"翻车"，本子直接进 #3 或 #4 文件夹。

### 2.4 写作启示（反向工程）

把申请书前 5-10 页打造成"三明治式快读包"：

- **首页**：题目 + 摘要 + 关键词 + 学科代码 → 让专家 60 秒内知道你做什么
- **第 2-3 页**：立项依据"漏斗 5 段式"前 2 段 + 1 张研究脉络图 → 让专家 5 分钟内同意你做的问题重要
- **第 4-5 页**：创新点 3 条 + 技术路线图 → 让专家 10 分钟内同意你的思路新颖、方案可行

后面的研究内容、可行性、研究基础是"支撑性精读材料"——专家在已经决定 A/B/C 的前提下，去这些章节找佐证或反驳。

## 三、会评（学科组评审）

### 3.1 评审主体

- 学科组由 NSFC 聘请的 **会评专家** 组成（通常每组 10-25 人，按二级 / 三级学科划分）
- 仅审议函评得分较高的项目（一般为函评总分前 30% 左右进入待评，前 12% 自动 I 类）
- 会评在北京集中召开（一般在 7-8 月），每个项目由 **主审 + 副审** 现场介绍并做"组内答辩"——但这里的"答辩"是会评专家间的讨论，**申请人不到场**

### 3.2 会评关注点（与函评显著不同）

会评专家更看重 **整体策略 + 边际差异**：

- **函评分数分歧大的项目**（如 3A2C）—— 主审需要解释为什么有 C，组内辩论
- **创新性"4A 是不是真的 4A"** —— 会评会检验"领域内的真实创新厚度"
- **学科组当年"指南方向"的匹配度** —— 重大研究计划尤其敏感
- **团队结构合理性** —— 重点 / 杰青 / 创新群体类必看
- **依托单位资源 / 平台是否真到位** —— 与现场口头核实
- **限项规则与重复申请** —— 同一个 PI 或同一个团队是否在多个本子里"撞车"

### 3.3 会评常见的 3 类决议

- 通过资助
- 不通过资助
- "降档"资助（如重点降为面上、杰青降为优青——较少见，但确实存在）

### 3.4 会评中"PI 表现"的 5 类隐性失分（来自公开讲座整合）

虽然申请人不在场，但会评专家会逐项目"复盘 PI 画像"。以下 5 类隐性失分常见：

1. **代表作"水分大"**：列出 10 篇 SCI，但仅 2 篇与课题直接相关——会评一句话就能拆穿。
2. **在研项目过多**：同时承担 4-5 个项目还来申请——"这个 PI 真的做得过来吗？"
3. **方向飘移**：5 年前做 A、3 年前做 B、现在又来申请 C——会评最忌讳"赶热点"型 PI。
4. **团队成员重叠**：共同申请人在另一个本子里也作为申请人——重复挂名直接扣分。
5. **过往结题质量差**：上一期 NSFC 项目结题评价"未完成 / 一般"或仅产出会议论文——会评会调档审视。

## 四、学部偏好差异（基于公开评审综述与公开讲座整合）

不同学部的评审文化、偏好有显著差异。以下基于《中国科学基金》刊发的各学部年度"基金评审工作综述"与多位评委公开分享归纳，**仅作大方向参考**。

### 4.1 F 学部（信息科学部）

- **偏好**：算法 / 系统 / 硬件的 **可量化创新**——明确写出与 SOTA baseline 的对比指标
- **重视**：开源代码 / 可复现性、benchmark 数据集
- **典型陷阱**：
  - "我们提出新算法"但没说和 baseline 比提升多少 → 创新性不清晰
  - 把"应用驱动 + 调参"当作创新 → 函评直接 C
  - 计算机科学 (F02) 与人工智能 (F06) 现已分开，**选错二级代码 = 评审专家不是小同行**
- **加分项**：1 张算法对比表（在哪些数据集上、哪些指标上、提升多少）

### 4.2 G 学部（管理科学部）

- **偏好**：理论贡献清晰 + 中国情境适配 + 实证设计严谨
- **重视**：识别策略（IV / DiD / RD / SCM 等）、内生性处理、外部效度
- **G01（管理科学与工程）**：偏数理建模 + 仿真 + 优化
- **G02（工商管理）**：偏理论建构 + 中国企业实证 + 案例研究
- **G03（经济科学）**：偏因果识别 + 政策评估 + 微观大数据
- **G04（宏观管理与政策）**：偏政策模拟 + 系统建模
- **典型陷阱**：
  - "讲了个好故事但没识别策略" → G03 直接 C
  - "理论贡献就是把 A 理论应用到 B 行业" → G02 视为伪创新
  - 用纯定性叙事申请实证类二级代码 → 学部偏好不匹配
- **加分项**：1 个清晰的因果识别图 + 1 段"为什么这个识别策略可信"

### 4.3 H 学部（医学科学部）

- **偏好**：清晰的科学问题（**机制 vs 临床转化** 二选一）+ 体外 / 体内 / 临床三层证据 + 预实验图
- **重视**：分子机制图、临床样本量计算、伦理审批
- **典型陷阱**：
  - "把临床观察当机制研究" → 创新性低
  - 缺预实验数据（H 学部对预实验数据的要求最严苛） → 可行性 C
  - 代表作是综述、Meta 分析为主 → 基础不匹配
- **加分项**：1 张机制示意图（信号通路 / 分子互作）+ 至少 1 组预实验 Western / qPCR / 测序数据

### 4.4 C 学部（生命科学部）

- **偏好**：原创性问题 + 模型生物适配 + 多组学方法
- **重视**：表型 → 机制 → 调控因子的逻辑链
- **典型陷阱**：用人源细胞系做"过度宣称的机制" → 评审会问"为什么不用模型动物"

### 4.5 E 学部（工程与材料）

- **偏好**：工程问题导向 + 物理机制清晰 + 可放大 / 可应用
- **重视**：实验装置图、理论计算与实验的呼应
- **典型陷阱**：仅做参数 sweep，不解释物理机制 → 创新性 C

### 4.6 共性差异速查表

| 学部 | 最看重 | 最致命 | 必备图 |
| --- | --- | --- | --- |
| A 数理 | 数学严谨性 / 物理深度 | 推导跳步 | 理论框架图 |
| B 化学 | 反应机理 / 路线设计 | 路线图缺合理性 | 合成路线图 |
| C 生命 | 表型-机制链 | 模型生物不匹配 | 机制示意图 |
| D 地球 | 数据 + 模型 + 野外验证 | 缺野外 / 卫星证据 | 时空分布图 |
| E 工材 | 物理机制 + 工程价值 | 仅参数 sweep | 装置 + 性能曲线 |
| F 信息 | SOTA 对比 + 可复现性 | 无 baseline 对比 | 算法对比表 |
| G 管理 | 因果识别 + 中国情境 | 缺识别策略 | 因果识别图 |
| H 医学 | 机制 + 转化 + 预实验 | 缺预实验图 | 信号通路图 |
| T 交叉 | 真交叉的科学问题 | 假交叉（拼凑） | 交叉问题图 |

## 五、加分项清单（评审专家"主动想给 A"的触发器）

| 加分点 | 写法示例 | 出现位置 |
| --- | --- | --- |
| 真正的预实验数据 | "本课题组前期数据见图 X，Y 现象在 N=20 中显著" | 可行性分析 |

| 清晰的技术路线图 | 节点 = 研究内容 R1-R5；连线 = 数据流；输出 = 论文 / 模型 / 数据 | 研究方案首页 |
| 明确的可证伪科学问题 | "本项目拟检验：在 X 满足 …… 条件时，Y 是否仍 ……" | 关键科学问题 |
| 与最新指南方向显式呼应 | "本项目对应 2026 年度 XX 学部'优先资助方向 No. N'" | 立项依据末段 |
| 已有合作产出 | "申请人与共同申请人已合作发表 3 篇 SCI（[1][2][3]）" | 团队介绍 |
| 国家 / 教育部重点平台 | "依托 XX 国家重点实验室（评估为优秀）" | 工作条件 |
| 1 张精炼的"机制示意图 / 概念框架图" | 论文常用风格的彩色概念图 | 立项依据末页 |

## 六、退稿信号（函评 D / E 的高频原因）

### 6.1 内容性退稿信号

- **创新性不清晰**：创新点写成"研究内容复述"
- **关键科学问题宽泛**：如"研究 XX 的影响因素"
- **方案空洞**：缺技术路线图、缺方法细节、缺预实验
- **代表作与课题脱节**：申请人 5 年代表作多在另一方向
- **预算异常**：设备费占比 > 50%、劳务费 = 0、版面费过高

### 6.2 形式性 / 学风性退稿信号（直接形式审查毙）

- **跨界跳跃 / 代码错配**：从生物突然申请管理科学；从计算机突然申请化学
- **抄袭嫌疑**：被检测出与公开来源（含已结题项目报告）大幅雷同；2026 年起 NSFC 对热点领域引入 AI 查重
- **限项违规**：在研项目数 + 申请数超出当年限项规定
- **个人贡献描述夸大**：2026 起明确禁止"虚假标注一作 / 通讯"
- **形式审查不通过**：模板不对、字号 / 行距修改、电子签名缺失

## 七、"会评中常被否决的 5 类 PR 表现"（公开讲座整合）

会评专家在讨论中容易"翻车"的 PI 画像：

1. **"摘要明星"型** —— 摘要写得漂亮，但研究内容章节与摘要脱节
2. **"代表作堆数"型** —— 列 10 篇 SCI 数量，但每篇都和课题擦边
3. **"赶热点"型** —— 5 年前做 A、3 年前换 B、今年又申请 C
4. **"借平台"型** —— 大量列依托单位的国家重点实验室，但代表作里看不到使用证据
5. **"团队挂名"型** —— 共同申请人是大牌但分工只写"指导咨询"，会评一眼看穿

## 八、写作 → 评审 反向工程的 3 层目标

把申请书写作目标设定为：

1. **30 秒内能让小同行答出**：你做什么？为什么有意义？（→ 题目 + 摘要 + 创新点首句）
2. **3 分钟能让小同行答出**：你的关键科学问题、研究方案、创新点（→ 立项依据漏斗 + 路线图）
3. **30 分钟能让小同行答出**：你能做成（基础匹配 + 可行性硬证据 + 预算合理）

如果上述任何一项做不到，就需要回炉重写对应章节。

## 九、2024-2026 改革要点（评审视角的影响）

NSFC 近年公开改革对评审视角的几个影响：

- **2020 起：分类申请改革** —— 申请人需在 4 类（→ 已调整为 2 类）"科学问题属性"中选 1 类，函评专家"按属性匹配评议标准"
  - **鼓励探索，突出原创**：评审更看"原创性"，对"完整性"宽容
  - **聚焦前沿，独辟蹊径**：评审看"前沿引领 + 切入角度"
  - **需求牵引，突破瓶颈**：评审看"问题真实性 + 应用价值"
  - **共性导向，交叉融通**：评审看"真交叉的科学问题"（不是拼凑）
- **2024-2025：评审专家"打招呼"专项整治** —— 评议透明度、申诉机制加强；评审专家匿名性更严
- **2026：申请书改版** —— 部分模板"破八股"，鼓励科研人员自主表达；但**评审 5 维核心未变**
- **2026：代表作"个人贡献描述"新增** —— 评审专家可逐篇查"你的真实贡献"，不能再"挂名通讯"

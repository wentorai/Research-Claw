# NSFC 申请书"报告正文"5 大模块写作模板

NSFC 申请书"报告正文"按公开模板分为 5 节（部分项目为 4 节）。本文件给出每节的**字数权重、写作要点、典型问题**。**字数总量和细节按当年模板为准**。

## 一、立项依据与研究内容（占报告正文约 60-70%）

包含 5 个子模块，是评审打分核心。

### 1.1 项目的立项依据

- **建议字数**：3000-5000 字（面上）；青年项目可压缩到 2000-3000 字
- **结构**：
  1. 研究意义（1 段）：为什么这个问题重要 — 学术价值 + 应用前景
  2. 国内外研究现状及发展动态（3-5 段）：按"代表性流派/方法"组织，**不要按时间编年**
  3. 对现状的批判性总结（1 段）：明确指出"已有研究尚未解决什么"
  4. 本项目研究问题（1 段）：自然引出
- **必备图**：建议放 1 张"研究脉络图"或"方法演进图"
- **常见问题**：写成文献综述、缺批判、研究问题未自然涌现

### 1.2 项目的研究内容、研究目标与拟解决的关键科学问题

- **建议字数**：1500-2500 字
- **结构**：
  1. 研究目标（3-5 句话）：限定边界、可检验
  2. 研究内容（3-5 个内容点，每点 2-3 段）：清晰编号 R1, R2, R3 ...
  3. 拟解决的关键科学问题（**2-3 个为宜**）：每个问题独立成段，含"为什么是关键科学问题"的论证
- **关键提示**：
  - 关键科学问题必须**可证伪**，避免"研究 XX 影响"这类表述
  - 研究内容与目标 **逐条对应**，避免"目标谈 A、内容做 B"
  - 关键科学问题 ≠ 研究内容；前者是"理论难点"，后者是"研究任务"

### 1.3 拟采取的研究方案及可行性分析

- **建议字数**：3000-5000 字
- **结构**：
  1. 总体研究方案与技术路线图（**必须放图**）
  2. 各研究内容的具体方案（与研究内容对应编号）：含数据来源、方法、计算/实验/模型设定
  3. 可行性分析（4 角度）：(a) 理论可行；(b) 方法可行（含预实验数据）；(c) 条件可行（设备、平台）；(d) 团队可行
- **常见问题**：技术路线图重复正文、可行性写成"我们有信心"、缺预实验数据

### 1.4 本项目的特色与创新之处

- **建议字数**：500-1000 字
- **结构**（建议 3 条，每条 1 段）：
  1. **科学思想 / 视角创新**（最重要）
  2. **方法 / 技术创新**
  3. **数据 / 体系 / 应用创新**
- **写法要点**：
  - 用"区别于已有……，本项目首次……"句式
  - 避免"本项目是首次研究 XX"这种空泛表述
- **常见问题**：把"研究内容"复述为"创新点"

### 1.5 年度研究计划及预期研究结果

- **建议字数**：500-1000 字
- **结构**：
  1. 年度计划（按年份分点，与研究内容对应）
  2. 预期成果：论文（数量 + 期刊层次）、专利、人才培养、学术交流
- **写法要点**：成果数量要"可达成"；论文期刊层次以"在 XX 类期刊发表论文 N 篇"表述

## 二、研究基础与工作条件（占 20-25%）

### 2.1 研究基础

- 简述申请人/团队近 5 年与本项目相关的代表作（5-10 篇）
- 每篇代表作只用 1-2 句话说"做了什么、与本项目的关系是什么"
- 避免列出与本项目无关的 SCI 数量

### 2.2 工作条件

- 实验室、大型仪器、计算平台、协作单位
- 国家重点实验室 / 教育部重点实验室等平台名称要写全

### 2.3 正在承担的与本项目相关的科研项目情况

- 列出正在承担的所有项目，**包括本次申请之外的任何 NSFC 项目**
- 必须主动说明"本项目与已承担项目的区别与联系"，否则会被扣"重复资助"分

### 2.4 完成国家自然科学基金项目情况（限近 5 年）

- 已结题 NSFC 项目的简要总结：完成情况、产出、与本项目关系
- 若申请人此前承担过 NSFC 项目却未结题或结题差，是重要的退稿信号

## 三、其他需要说明的问题

- 申请人与依托单位过往违规情况；同一年其他申请的关联说明；伦理审批情况
- 一般留 100-300 字即可；无相关问题写"无"

## 总体提示

- "报告正文"建议总字数控制在 1.5-3 万字（面上 / 重点）；青年项目 1-1.5 万字
- 务必使用 NSFC 系统提供的模板，不可自行调整字号、行距、标题层级
- 预算与预算说明书在另一表单中提交，写作时要与"研究内容 / 研究方案"互相印证

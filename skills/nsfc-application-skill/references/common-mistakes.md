# NSFC 申请书高频被毙原因（深度版：15 类内容性 + 5 类隐性 PI 失分）

本文件列出 NSFC 申请书最常见的 **15 大类内容性退稿原因** 与 **5 类隐性 PI 失分点**，每类配 fix 方案。多数失败申请书并非"想法不好"，而是"表述未让小同行看出好"。

> 本文件基于 NSFC 公开发布的"申请书撰写要点"、各学部公开的"年度评审工作综述"（《中国科学基金》期刊），以及多位获资助专家在公开讲座中的分享整合而成。比例为经验性估计，不代表官方统计。各学部 / 学科退稿原因分布略有差异。

---

## A. 内容性退稿原因（15 类）

### 一、创新性表述失焦（最常见，占退稿 25%+）

**典型症状**：
- 创新点写成"研究内容的复述"
- 用"首次"+"系统性"+"全面"等空洞修饰词
- 把"新数据"或"新案例"当作创新

**示例（劣）**：
> 创新点：本项目首次系统性研究了某某地区的某某现象，构建了完整的理论框架。

**Fix**：
- 用 "**区别于已有 ……，本项目 ……**" 句式，必须显式对比
- 创新点 ≤ 3 条；分别对应 思想 / 方法 / 应用
- 每条创新点 1 段（5-8 行）；首句给"总结性结论"
- 推荐"3+1 写法"：3 条主创新 + 1 个项目级"概念命名"

### 二、关键科学问题宽泛、不可证伪（占退稿 15%+）

**典型症状**：
- 关键科学问题表述为"研究 X 的影响因素"、"探讨 X 的内在规律"
- 关键科学问题与研究内容混为一谈
- 提了 5-6 个问题，每个都泛泛

**Fix**：
- 关键科学问题 **2-3 个为宜**，每个独立成段
- 每个问题应满足"**有理论难度 + 可证伪 + 可操作**"
- 用 3 类范式句式之一：
  - **范式 A（机制类）**："在 Z 情境下，X 与 Y 之间的 W 机制是什么？"
  - **范式 B（条件类）**："在什么条件下，X 是否仍 Y？"
  - **范式 C（识别类）**："如何识别 X 对 Y 的因果效应？"
- 关键科学问题 ≠ 研究内容；前者是"为什么难"，后者是"我做什么"

### 三、研究方案空洞、缺技术路线图（占退稿 15%+）

**典型症状**：
- 方案章节写成"将采用 XXX 方法 + XXX 数据"，无方法细节
- 没有技术路线图，或图为简单流程框（5 个矩形横向排列）
- 可行性写成"我们有信心、团队经验丰富"

**Fix**：
- **必须放 1 张技术路线图，含 6 要素**：KP 节点、R 节点、数据流标签、主路径 / 备选路径、里程碑、输出节点
- 每个研究内容 1-2 段方法细节：含数据 / 模型 / 实验设计 / 显著性判据
- 可行性分析按 4 角度展开（"3 段式（人 / 物 / 法）+ 理论可行"）
- 至少有 1 处"预实验图 / 数据"，作为可行性硬证据

### 四、申请人基础与课题脱节（占退稿 10%+）

**典型症状**：
- 申请人代表作集中在 A 方向，本项目却投 B 方向
- 列出大量与课题无关的 SCI 数量
- "研究基础"章节像"个人简历"

**Fix**：
- 代表作精选 5-10 篇，**全部与课题相关**
- 每篇用 1-2 句话说"做了什么 + 与本项目研究内容 R? 的关系"
- 若申请人确有方向调整，需用 1 段说"为什么本项目是顺势延伸"，并给出过渡性成果
- **2026 起**：每篇代表作必须填"个人贡献描述"——不能写"我是第 N 作者"，要写"我设计了 X 实验 / 撰写了 Y 部分"

### 五、预算与研究内容不匹配（占退稿 8%+）

**典型症状**：
- 预算明显超标（设备费占 50%+）
- 预算明显压低（材料费 1 万、劳务费 0）
- 国合项目无国际合作交流费
- 研究内容里说"开展 5 次田野调查"，差旅费却不够

**Fix**：
- 写完研究方案，**反向倒推预算**
- 设备费控制在 ≤ 30%（青年项目 ≤ 5%）
- 劳务费按"研究生数 × 月津贴 × 月数"估算
- 包干制项目仍要在"经费说明"中给出大致结构

### 六、立项依据写成"科普 + 文献流水账"（占退稿 8%+）

**典型症状**：
- 段 1 像"高中生的科普介绍"
- 段 2-4 按时间编年（"2010 年 A 提出 …… 2015 年 B 提出 ……"）
- 缺批判性总结
- 研究问题"突兀给出"，与现状脱节

**Fix（漏斗 5 段式）**：
- 段 1（领域 + 重要性）：100 字内答出"这个问题对学科 / 国家需求精确意味着什么"
- 段 2-4（每段 1 个流派）：按流派组织，**段首句直接概括流派**，末句给批判性局限
- 段 5（批判性总结 + 引出问题）：研究问题与"已有缺口"完全对位

### 七、关键科学问题"撒胡椒面"（占退稿 5%+）

**典型症状**：列出 5-6 个关键科学问题，每个都是"研究 X 与 Y 的关系"。

**Fix**：
- 关键科学问题 **2-3 个为宜**——多了显得不聚焦
- 每个 KP 独立成段（200-400 字），含"问题陈述 + 为什么是关键科学问题 + 本项目如何切入" 3 段式

### 八、技术路线图无信息量（占退稿 5%+）

**典型症状**：
- 5 个矩形横向排列，箭头一串
- 路线图与正文研究内容编号不对应
- 缺数据流 / 缺判据 / 缺备选路径

**Fix**：必须含 6 要素（见上文"研究方案空洞"）。**正文紧跟必须有 200-400 字"文字描述"**——评审有时只读文字。

### 九、缺预实验数据（H / E 学部尤其致命，占退稿 5%+）

**典型症状**：可行性分析全是"我们将 …… 我们计划 ……"，没有任何"我们已经做出了 ……"的硬证据。

**Fix**：
- 至少 1 张预实验图（**图注必须注明"申请人课题组前期数据"**）
- 1 段实验数据描述（N 多大、p 多少、效应量多少）
- 1 处对应"已发表论文 / 已申请专利 / 已开源代码仓库"的引用

### 十、研究目标不可检验 / 边界不清（占退稿 4%+）

**典型症状**：
- 研究目标写成"全面深入研究 ……"
- 缺成功 / 失败判据
- 目标与研究内容不对应

**Fix**：
- 研究目标 3-5 句话；每句必须可检验
- 每个研究内容 R 必须给"成功判据"（如"显著性 p<0.05、效应量 > 0.3"）

### 十一、年度计划与研究内容脱节（占退稿 3%+）

**典型症状**：年度计划写成"第 1 年开题、第 2 年实验、第 3 年总结"——和研究内容编号无关。

**Fix**：年度计划按 R1-R5 编号，**每个 R 标注 M1-M3 里程碑**。

### 十二、预期成果"过度承诺"（占退稿 3%+）

**典型症状**：面上项目承诺 10 篇 1 区论文、3 项发明专利。

**Fix**：
- 论文层次以"在 XX 类期刊发表论文 N 篇"为表述
- 数量"可达成"（面上 4-8 篇 SCI 较合理）
- 3 类成果：A 类（学术）+ B 类（应用）+ C 类（人才培养与交流）

### 十三、引用与文献"自引过多 / 引用陈旧"（占退稿 3%+）

**典型症状**：
- 自引占比 > 30%
- 引用文献最新的也是 5 年前的

**Fix**：
- 引用文献 70% 以上为近 5 年；30% 以上为近 2 年
- 自引控制在 20% 以内
- 至少引用 3-5 篇本领域顶级期刊近 2 年文献，**显示"对前沿熟悉"**

### 十四、国合 / 重大类项目"合作必要性"不清（占退稿 2%+）

**典型症状**：写"双方合作多年"，但说不清"为什么必须合作 + 双方互补性 + 联合产出"。

**Fix**：
- 1 段"合作必要性"：写"中方有 X 优势、外方有 Y 优势、合作产生 Z 价值"
- 1 段"双方分工"：写明"中方做 R1, R3、外方做 R2, R4"
- 1 段"联合产出"：合著论文、联合培养、互访方式

### 十五、写错申请代码（直接形式审查毙）

**典型症状**：
- 选错二级代码，函评派送给"不是小同行"
- 跨学部跳跃（生物 → 管理）

**Fix**：
- **每年核对 NSFC 当年代码表**——代码改革后多有调整
- 关键词 5-7 个，覆盖二级 / 三级代码
- 跨学部时考虑交叉科学部 (T)，但要明确"交叉的科学问题是什么"

## B. 隐性 PI 失分点（5 类）

会评专家在讨论中容易"翻车"的 PI 画像。这些失分点**不会显式出现在评审意见里**，但会大幅拉低本子在会评中的"主审 / 副审推荐"。

### B-1. "代表作堆数"型

**症状**：列 10 篇 SCI，但仅 2 篇与课题直接相关。

**会评心理**：拆穿"申请人在凑数量"。

**Fix**：
- 代表作精选 5-10 篇，**全部与课题相关**
- 每篇 1-2 句"与本项目研究内容 R? 的关系"
- **2026 起**：每篇必须填"个人贡献描述"——具体说核心问题 / 路线设计 / 实验设计 / 撰写

### B-2. "在研项目过载"型

**症状**：同时承担 4-5 个项目还来申请——评审会问"这个 PI 真的做得过来吗？"

**Fix**：
- 在"正在承担的科研项目情况"中**全部列出**（含非 NSFC 的省部级 / 横向）
- 主动说明"本项目与已承担项目的区别和联系"
- 若在研过多，建议先结题再申请

### B-3. "方向飘移"型

**症状**：5 年前做 A、3 年前做 B、今年又申请 C——评审最忌讳"赶热点"型 PI。

**Fix**：
- 用 1 段说"为什么本项目是 5 年方向的逻辑延伸"
- 给出"过渡性成果"：5 年内已经在 B → C 转向中发表的论文
- 杰青 / 优青范式：在立项依据段 1 加"申请人 5 年聚焦 …… 方向"

### B-4. "团队挂名 / 借平台"型

**症状**：
- 共同申请人是大牌但分工只写"指导咨询"
- 大量列依托单位的国家重点实验室，但代表作里看不到使用证据

**Fix**：
- 团队每位成员的"分工"必须具体（不写"协助申请人完成所有内容"）
- 国家重点实验室列出后，要在"研究基础"中显式呼应（"代表作 [3] 主要使用本实验室的 …… 仪器"）
- 共同申请人与申请人**已合作发表 ≥ 1 篇 SCI** 是会评的隐性门槛

### B-5. "过往结题质量差"型

**症状**：上一期 NSFC 项目结题评价"未完成 / 一般"或仅产出会议论文。

**Fix**：
- 在"完成国家自然科学基金项目情况"中**主动说明改进措施**
- 强调"本项目与上期项目的区别"（避免"重复资助"嫌疑）
- 上期产出在本项目中如何"作为基础" → 转化为加分

## C. 其他常见小毛病（致命度低但累计扣分）

- **摘要不点题**：摘要前 2 句没说"做什么 + 为什么有意义"
  - **Fix**：摘要前 2 句必须答出 W1（做什么）+ W2（为什么重要）
- **图表过多 / 过少**：图过多看着花、过少看着空
  - **Fix**：建议 4-8 张高质量图，每张图必有图注 + 在正文有引用
- **格式不规范**：未使用 NSFC 模板；字号 / 行距 / 标题层级随意改动
  - **Fix**：用 ISIS 系统下载的当年模板，**不可自行调整**
- **学风诚信问题**：抄袭、重复申请、未结题历史项目
  - **Fix**：申请书必须为申请人原创；2026 起 NSFC 对热点领域引入 AI 查重
- **个人贡献夸大**：2026 起明确禁止"虚假标注一作 / 通讯"
  - **Fix**：如实标注；共同一作 / 共同通讯按"共同 N 作 / 共同 N 通讯"标注
- **限项违规**：在研项目数 + 申请数超出当年限项规定
  - **Fix**：申报前查阅当年《项目指南》"限项申请规定"

## D. 自审清单（写完后逐项核对）

### D-1. 内容核对

- [ ] 摘要 200-400 字，前 2 句答出"问题 + 意义"
- [ ] 立项依据按"漏斗 5 段式"，含批判性总结
- [ ] 关键科学问题 2-3 个，每个可证伪、用了 3 类范式句式之一
- [ ] 研究内容 R1-R5 编号清晰、与 KP 显式对应
- [ ] 技术路线图含 6 要素（KP / R / 数据流 / 主备路径 / 里程碑 / 输出）
- [ ] 路线图后有 200-400 字文字描述
- [ ] 可行性分析含"3 段式（人 / 物 / 法）+ 理论可行"4 角度
- [ ] 至少 1 处预实验数据 / 图（图注注明"申请人课题组前期数据"）
- [ ] 创新点 ≤ 3 条 + 1 个概念命名（"3+1 写法"）
- [ ] 创新点避免"首次 / 填补 / 国际领先"等空泛词
- [ ] 申请人代表作 5-10 篇，全部与课题相关 + 个人贡献描述
- [ ] 预算与研究内容互相印证

### D-2. 形式核对

- [ ] 申请代码（二级 / 三级）已用当年代码表核对
- [ ] 限项规定已核对
- [ ] 学风诚信无问题（无重复申请、无抄袭嫌疑）
- [ ] 关键词 5-7 个，覆盖二级 / 三级代码方向
- [ ] 已让 1-2 位小同行预审

### D-3. 隐性 PI 失分点核对

- [ ] 代表作"水分"低（不是凑数量）
- [ ] 在研项目数合理（不过载）
- [ ] 方向连续（无明显飘移）
- [ ] 团队分工具体（无挂名）
- [ ] 过往结题质量好（如有差，主动说明改进）

## E. 注意事项

- 本文件总结的高频原因来自 NSFC 公开公布的"申请书撰写要点"和大量公开评审反馈分享，**比例为经验性估计**，不代表官方统计
- 各学部 / 学科退稿原因分布略有差异：工科偏可行性、理科偏创新性、人文社科偏问题清晰度
- "隐性 PI 失分点"主要在会评阶段发挥作用——会评不出现在评审意见里，但决定"主审 / 副审推荐力度"
- **最终请以当年 NSFC 官网公布的《申请书撰写要点》《项目评审办法》《项目指南》为准**
